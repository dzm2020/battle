在战斗服务器的技能系统中，目标选取类型 通常按 作用范围、阵营关系、选取规则 三个维度划分。

以下是游戏开发中最常见的分类（含设计要点）：

一、按「作用范围」分类（最核心）
类型	说明	示例
单体	只影响 1 个目标	火球术、治疗术
自身	只影响施法者自己	疾跑、护盾
扇形	施法者前方扇形区域	横扫、龙息
圆形 / 范围	以目标点为中心的圆	暴风雪、手雷
矩形 / 直线	直线路径上的敌人	闪电链、穿刺
全屏 / 全场	影响战场上所有单位	全屏 AOE、群体复活
链式	在多个目标间跳跃	闪电链、弹射治疗
随机	从合法目标中随机 N 个	随机射击、混乱箭
二、按「阵营关系」分类
类型	说明
敌方	只能选敌人（通常要求是敌对阵营）
友方	只能选队友（包括自己或不包括自己）
自身	仅自己
全部	无视阵营（如中立单位、所有活体）
指定阵营	红方 / 蓝方 / 怪物 / 召唤物等
三、按「选取规则 / 过滤条件」分类
规则	说明
最近	距离施法者最近的目标
最远	距离最远的目标
血量最低	常用于治疗或斩杀
血量百分比最低	更智能的群体治疗
攻击力最高	优先控制高威胁目标
仇恨最高	坦克拉怪逻辑
有 Buff/Debuff 者	如驱散中毒、延长流血
无特定状态	如“没有护盾的目标”
玩家优先 / 怪物优先	PvP / PvE 混合场景
召唤者	召唤物技能指向主人
四、按「交互方式」分类（玩家操作）
类型	说明
无目标	不需选目标，直接释放（如自我增益）
当前目标	对玩家当前锁定的目标释放
鼠标指向	对鼠标悬停的单位释放（MMO 常见）
地面位置	选择地面坐标释放范围技能
智能施法	没有目标时自动找合法目标
弹窗确认	回合制游戏中选择目标列表
五、典型组合示例（技能配置）
一个技能往往由 范围 + 阵营 + 规则 组合而成

技能示例	范围	阵营	规则
群体治疗	圆形	友方	血量百分比最低的 5 个
顺劈斩	扇形	敌方	前方 180° 所有敌人
复仇之怒	自身	自身	无
闪电链	链式	敌方	优先未被链过的目标
斩杀	单体	敌方	血量最低
战复	单体	友方	死亡单位

七、常见陷阱（服务器开发注意）
问题	说明
性能	每帧全量遍历所有单位查找目标 → 用空间分区（网格/四叉树）
死亡单位	技能释放瞬间目标死亡 → 需要重新选或 MISS
隐身/无敌	必须不可选中的状态应提前过滤
范围快照	技能引导中目标走出范围 → 仍然命中（或打断）需明确规则
归属权	AOE 伤害需要记录每个目标的伤害来源（用于仇恨/统计）
