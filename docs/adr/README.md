# 架构决策记录（ADR）

本目录存放 **Architecture Decision Records**：记录「为什么这样设计」，避免口头约定丢失。

## 何时撰写

- 改变包边界、公共 API、并发/线程模型、配置加载语义、持久化或网络协议。
- 存在多个可行方案且取舍非显而易见。

## 命名

- `NNN-short-title.md`（三位序号，全小写英文与短横线）。
- 文内建议包含：背景、决策、备选方案、后果、状态（提议 / 已采纳 / 已废弃）。
